# Contributing to RetailManager

## Getting Started

1. Fork the repository
2. Clone your fork
3. Create a feature branch

## Development Setup

See [Local Development](docs/local-development.md) for setup instructions.

## Code Standards

### Backend (Java/Spring)
- Use meaningful variable names
- Add JavaDoc for public methods
- Follow Spring conventions
- Write unit tests for new features

### Frontend (Angular/TypeScript)
- Use strict TypeScript (no `any`)
- Follow Angular style guide
- Use proper naming conventions
- Write tests for components

## Pull Request Process

1. Update documentation if needed
2. Ensure code compiles and tests pass
3. Update the CHANGELOG if applicable
4. Request review from maintainers

## Security

- Never commit secrets or credentials
- Use environment variables for sensitive data
- Follow security guidelines in [docs/security.md](docs/security.md)

## Questions

Open an issue for questions about contributing.
