# Global Agent Configuration

## 📍 Monorepo Overview

RetailManager is a **monorepo** with:
- **2 Angular frontends** (e-commerce B2C + web-client B2B)
- **Spring Boot microservices** backend
- **Shared skills** for UI design, accessibility, and SEO

## 📂 Project Structure & Guides

Each project has **AGENTS.md** with specific guidelines:

### Frontend - E-commerce
- **Path**: `frontend/ecommerce/`
- **Guide**: `frontend/ecommerce/AGENTS.md`
- **Code Standards**: `frontend/ecommerce/.instructions.md`
- **Stack**: Angular 21, TypeScript, Vanilla CSS, Standalone components
- **Tasks**: Product catalog, shopping cart, checkout, user profiles

### Frontend - Web Client
- **Path**: `frontend/web-client/`
- **Guide**: `frontend/web-client/AGENTS.md`
- **Stack**: Angular 21, TypeScript, Tailwind CSS, SSR, Standalone components
- **Tasks**: Admin dashboard, B2B portal, inventory management, analytics

### Backend - Microservices
- **Path**: `backend/erphub-api/microservices/`
- **Guide**: `backend/AGENTS.md`
- **Stack**: Spring Boot 4, Java 25, Domain-driven design, Event-driven
- **Tasks**: REST APIs, database schema, event communication, service scaling

## 🔄 Integration Patterns

### Frontend → Backend
- All API calls go through Gateway: `http://localhost:8080`
- Use `/api/` prefix for all endpoints
- Microservices are transparent (routed by gateway)

### Project-Specific Integration

**E-commerce** integrates with:
- `product-sv` for product data
- `sales-sv` for orders
- `inventory-sv` for stock
- `customer-sv` for user profiles

**Web Client** integrates with:
- All microservices (can aggregate data)
- Primarily: product, inventory, analytics, reporting

**Microservices** communicate via:
- REST calls through Gateway
- RabbitMQ events for async
- Separate databases (no cross-service DB access)
