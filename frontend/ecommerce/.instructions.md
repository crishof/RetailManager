# RetailManager E-commerce Frontend

## Tech Stack

- **Framework**: Angular 21.2 (standalone components)
- **Language**: TypeScript 5.9 (strict mode)
- **Package Manager**: npm 11.13
- **Testing**: Vitest
- **CSS**: Vanilla CSS (modular by component)

## Project Guidelines

### Code Style

- **Strict TypeScript**: All files must compile with `noImplicitAny`, `strict`, `noImplicitReturns`
- **Standalone Components**: Use `@Component()` with `imports` array, not NgModules
- **Reactive Programming**: Use Signals for state, computed() for derived state
- **Change Detection**: Always use `ChangeDetectionStrategy.OnPush`

### Component Patterns

```typescript
// ✅ Correct: Standalone with OnPush
@Component({
  selector: 'app-product-card',
  imports: [CommonModule, NgOptimizedImage],
  changeDetection: ChangeDetectionStrategy.OnPush,
  template: `...`,
  styles: [`...`]
})
export class ProductCardComponent {
  product = input<Product>();
  selected = output<Product>();
}

// ❌ Wrong: NgModules, HostBinding, template logic
```

### Accessibility

Every component MUST:
- Pass AXE accessibility checks
- Meet WCAG AA color contrast (minimum 4.5:1)
- Support keyboard navigation
- Include semantic HTML and ARIA attributes

## Commands

```bash

npm run build          # Production build
npm run watch          # Watch mode
npm test               # Run tests
```

## Backend Integration

API Gateway: `http://localhost:8080`

Key endpoints (via Gateway):
- `GET /api/products` (product-sv)
- `POST /api/orders` (sales-sv)
- `GET /api/cart` (cart-sv if exists)

## Important Notes

1. **No Tailwind CSS**: Using vanilla CSS for now (can be added later)
2. **Standalone Approach**: Components import dependencies directly
3. **Signals over RxJS**: Prefer Angular signals for local state
4. **Lazy Loading**: Use `canActivate()` guards and route-level code splitting
5. **Tree-shaking**: Import only what you need from Angular
